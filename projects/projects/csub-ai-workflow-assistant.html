CSUB AI Workflow Assistant

A full-stack application developed for California State University, Bakersfield to simplify student hiring workflows, policy questions, form selection, approval routing, and grant information lookup.

Full-Stack Development Artificial Intelligence Cloud Computing Serverless Architecture Real Client Project
Project Overview

Simplifying complex university processes

The CSUB AI Workflow Assistant is a full-stack web application developed for California State University, Bakersfield through the AI Summer Camp at Cal Poly.

The project was designed to help university staff navigate student hiring procedures, identify the correct forms, understand approval routing, answer policy questions, and access grant information from one application.

Rather than creating a general-purpose chatbot, the application combines guided workflows, deterministic business logic, and retrieval-based artificial intelligence. This architecture gives users flexibility while maintaining accuracy in areas where administrative decisions must follow defined university rules.

Challenge & Solution

Building accuracy into an AI-assisted workflow

The system needed to make administrative information easier to access without allowing generative AI to independently make decisions that should follow established policies.

⚠️

The Challenge

University hiring processes involve multiple forms, funding sources, routing paths, grant records, checklists, and policy documents. Staff may need to search several locations before determining the correct procedure for a single hiring request.

💡

The Solution

The application brings these resources into one guided experience. Deterministic logic handles defined hiring and grant workflows, while Amazon Bedrock answers documentation questions using a Knowledge Base created from official university sources.

Core Features

One application, multiple administrative tools

01

Guided Hiring Workflows

Staff are guided through hiring type, funding source, form selection, approval routing, grant lookup, and required checklist steps.

02

Deterministic Decision Logic

Defined university rules are processed using controlled application logic instead of depending on unpredictable AI responses.

03

Grounded Policy Answers

Amazon Bedrock responds to policy and documentation questions using a Knowledge Base built from official CSUB documents.

04

Grant Information Lookup

Users can locate relevant fund numbers, project numbers, principal investigators, and grant analyst information.

05

Automated Document Processing

Newly uploaded documents are processed automatically so the assistant can use current information without requiring a new software deployment.

06

Administrative Summary

At the end of a workflow, the system presents a clear summary of the selected form, funding source, routing information, and hiring requirements.

System Architecture

A hybrid full-stack and serverless architecture

The architecture separates the user interface, API layer, deterministic workflow logic, AI services, and automated document ingestion process.

User Workflow

The Next.js application receives user selections and questions, then securely sends requests through API routes to backend services.

AI Workflow

AWS Lambda communicates with Amazon Bedrock, which searches the official document Knowledge Base before generating a grounded response.

Document Workflow

New documents uploaded to Amazon S3 trigger processing through AWS Lambda and Amazon Textract before being added to the Knowledge Base.

Application Interface

Designed around clear, guided interactions

The interface avoids presenting users with an empty chatbot. Instead, it provides visible workflow stages, structured choices, progress indicators, and a final administrative summary.

Automated Document Pipeline

Keeping the assistant current without code changes

01

Administrator uploads a document

Updated university policies, procedures, and reference documents are uploaded to a designated Amazon S3 bucket.

02

AWS Lambda detects the update

The upload event automatically triggers a serverless processing workflow without requiring manual intervention.

03

Amazon Textract processes scanned content

When documents contain scanned pages, Amazon Textract extracts the text and structured information needed for indexing.

04

The Knowledge Base is updated

Processed content becomes available to Amazon Bedrock so future responses can use the newest official information.
