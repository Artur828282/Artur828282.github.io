CSUB AI Workflow Assistant

A full-stack application developed for California State University, Bakersfield to simplify student hiring workflows, policy questions, form selection, approval routing, and grant information lookup.

Full-Stack Development Artificial Intelligence Cloud Computing Serverless Architecture Client Project
Project Overview

Simplifying complex university processes

The CSUB AI Workflow Assistant is a full-stack web application developed for California State University, Bakersfield through the AI Summer Camp at Cal Poly.

The project was created to help university staff navigate student hiring procedures, identify the correct forms, understand approval routing, answer policy questions, and access grant information from one application.

Instead of relying on a generic chatbot, the application uses a hybrid architecture. Guided hiring workflows and grant lookups use deterministic logic, while policy and documentation questions are answered using Amazon Bedrock and a Knowledge Base built from official university documents.

Client California State University, Bakersfield
Program AI Summer Camp at Cal Poly
Project Type Full-stack cloud application
Primary Users University administrative staff
Challenge & Solution

Building accuracy into an AI-assisted workflow

The application needed to make administrative information easier to access without allowing generative AI to independently make decisions governed by university policies.

⚠️

The Challenge

University hiring processes involve multiple forms, funding sources, routing paths, grant records, checklists, and policy documents. Staff may need to search several locations before identifying the correct procedure.

💡

The Solution

The system combines guided workflows with retrieval-based AI. Deterministic logic handles defined hiring and grant decisions, while Amazon Bedrock answers policy questions using trusted university documentation.

Core Features

One application, multiple administrative tools

01

Guided Hiring Workflows

Guides staff through hiring type, funding source, form selection, approval routing, grant lookup, and required checklist steps.

02

Deterministic Decision Logic

Defined university rules are handled using controlled application logic rather than unpredictable generated responses.

03

Grounded Policy Answers

Amazon Bedrock answers questions using a Knowledge Base built from official CSUB policy and hiring documents.

04

Grant Information Lookup

Users can locate fund numbers, project numbers, principal investigators, and grant analyst information.

05

Automated Document Updates

Newly uploaded documents are processed automatically so the system can use updated information without redeployment.

06

Administrative Summary

The completed workflow provides a clear summary of the selected form, funding source, routing information, and hiring requirements.

System Architecture

A hybrid full-stack and serverless architecture

The architecture separates the user interface, API layer, deterministic workflow logic, AI services, and automated document processing pipeline.

User Workflow

The Next.js application receives questions and selections, then sends requests through API routes to backend services.

AI Workflow

AWS Lambda communicates with Amazon Bedrock, which searches the official document Knowledge Base before generating an answer.

Document Workflow

Documents uploaded to Amazon S3 trigger AWS Lambda and Amazon Textract processing before being added to the Knowledge Base.

Application Interface

Designed around clear, guided interactions

Instead of presenting an empty chatbot interface, the application provides visible stages, structured choices, progress indicators, and a final administrative summary.

Automated Document Pipeline

Keeping the assistant current without code changes

01

Upload to Amazon S3

Administrators upload updated university policies, procedures, and reference documents to a designated S3 bucket.

02

AWS Lambda detects the update

The upload automatically triggers a serverless processing workflow without requiring manual intervention.

03

Amazon Textract extracts document content

Textract extracts text and structured information from scanned documents when standard text extraction is insufficient.

04

The Knowledge Base is updated

The processed content becomes available to Amazon Bedrock so future responses can use the latest official information.
