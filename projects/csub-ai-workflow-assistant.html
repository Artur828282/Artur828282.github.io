CSUB AI Workflow Assistant

A full-stack cloud application developed for California State University, Bakersfield to simplify student hiring workflows, locate forms, answer policy questions, and provide grant information from one application.

Project Overview

The CSUB AI Workflow Assistant is a full-stack web application developed for California State University, Bakersfield through the AI Summer Camp at Cal Poly.

The application was created to simplify the student hiring process by helping university staff navigate hiring workflows, determine which forms are required, understand approval routing, answer policy questions, and access grant information from a single location.

Project Challenge

University hiring processes can involve multiple forms, funding sources, approval paths, grant records, checklists, and policy documents. Staff may need to search through several resources before identifying the correct procedure for a single hiring request.

The goal was to create a system that made this information easier to access while maintaining accuracy and ensuring that important administrative decisions remained grounded in official university procedures.

Hybrid System Design

Instead of relying entirely on a general-purpose chatbot, the application uses a hybrid architecture. Guided hiring workflows and grant lookups use deterministic application logic, allowing the system to provide consistent results based on established rules.

Policy and documentation questions are handled using Amazon Bedrock with a Knowledge Base built from official university documents. This allows the AI component to retrieve information from trusted sources rather than responding only from general model knowledge.

Core Features

Guided Hiring Workflows

Guides staff through hire type, funding source, required forms, approval routing, and checklist requirements.

Grant Information Lookup

Provides relevant fund numbers, project numbers, principal investigators, and grant analyst information.

Grounded Policy Answers

Uses Amazon Bedrock and official university documentation to answer policy and procedural questions.

Administrative Summary

Produces a clear summary showing the recommended form, funding source, routing information, and next steps.

Automated Document Processing

Processes newly uploaded university documents and adds their information to the AI Knowledge Base.

Expandable Workflow Platform

The application was structured so additional administrative workflows can be added as future modules.

System Architecture

The application separates the user interface, API routes, backend processing, AI services, and document update workflow. The Next.js frontend receives user questions and selections, while AWS Lambda handles backend requests and communicates with Amazon Bedrock.

Amazon Bedrock searches the Knowledge Base containing official CSUB documents before generating policy-based answers. Guardrails are also used to support safer and more responsible responses.

Application Interface

The interface was designed around clear, guided interactions rather than presenting users with only an empty chat box. Visible workflow steps, structured choices, status indicators, and final summaries help staff understand where they are in the process.

Application Dashboard

The main dashboard provides access to student hiring and future administrative workflow modules.

Guided Hiring Workflow

A step-by-step process helps users determine the correct hiring type, funding source, forms, and approval route.

Hiring Summary

The completed workflow organizes essential hiring and grant information into one administrative summary.

Cloud Architecture

The system connects the web application, API routes, serverless backend, AI services, and document pipeline.

Automated Document Update Pipeline

The application includes an automated process for keeping the assistant current when university documentation changes.

1

Upload to Amazon S3

An administrator uploads an updated university document to a designated Amazon S3 bucket.

2

AWS Lambda Trigger

The new upload automatically triggers a serverless processing workflow using AWS Lambda.

3

Amazon Textract Processing

Amazon Textract extracts text and structured information from scanned documents when necessary.

4

Knowledge Base Update

The processed content is added to the Amazon Bedrock Knowledge Base so the assistant can use the latest official information without code changes or redeployment.
